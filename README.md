# AI-based Classification of Germline and Somatic DNA Mutations

## Overview

This project investigates whether DNA mutations can be classified as Germline or Somatic using only sequence information and biologically relevant contextual features.

Several machine learning and deep learning approaches are evaluated, ranging from traditional k-mer models to transformer-based genomic embeddings and hybrid models that combine sequence information with biological knowledge.

The objective is to assess how much information is contained within the local DNA sequence surrounding a mutation and whether additional biological features improve classification performance.

---

## Dataset

The dataset was created by combining mutations from:

- ClinVar (Germline mutations)
- COSMIC (Somatic mutations)

For each mutation:

1. The mutation position was mapped to the human reference genome (GRCh38).
2. A DNA sequence window centered on the mutation was extracted.
3. Wild-type and mutated sequences were generated.
4. Biological annotation features were added.

### Target Classes

| Class | Description |
|---------|-------------|
| 0 | Germline Mutation |
| 1 | Somatic Mutation |

---

## Data Cleaning

To avoid information leakage:

- Duplicate mutations were removed.
- Ambiguous sequences appearing in both classes were excluded.
- Dataset splits were performed at sequence level.

### Train / Validation / Test Split

- Train: 60%
- Validation: 20%
- Test: 20%

Stratified sampling was used to preserve class balance.

---

## Feature Engineering

### Sequence Features

The mutation sequence window was transformed into numerical representations using:

- Character n-grams
- K-mer frequencies
- Mutation-centered subsequences

### Biological Features

Additional mutation descriptors include:

- Reference allele
- Alternative allele
- Chromosome
- Mutation type
- Sequence composition metrics

---

# Models Evaluated

## 1. Logistic Regression

Baseline model using k-mer frequency features.

## 2. XGBoost

Gradient boosted decision trees trained on engineered sequence features.

## 3. XGBoost Using Mutation-Centered Windows

Evaluates whether immediate mutation context alone is sufficient for classification.

## 4. 1D Convolutional Neural Network (CNN)

Deep learning model trained directly on one-hot encoded DNA sequences.

## 5. Nucleotide Transformer + XGBoost

Uses embeddings generated by a pretrained genomic foundation model.

## 6. Hybrid Model

Combines:

- Nucleotide Transformer embeddings
- Biological mutation descriptors

to classify mutations as Germline or Somatic.

---

## Model Evaluation

Models are evaluated using:

- Accuracy
- ROC-AUC
- Precision
- Recall
- F1-Score
- Matthews Correlation Coefficient (MCC)

MCC is used as the primary metric due to its robustness to class imbalance and its widespread use in bioinformatics classification tasks.
